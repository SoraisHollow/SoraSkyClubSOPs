Moderation Logging SOP (Bouncers & Security)

This page explains how Bouncers and Security should log incidents, record evidence, and prepare clean case files in Notion for Admin review.

Scope: This SOP applies to Bouncers, Security, and other staff who handle moderation actions. Admins and Owners review and finalize cases after you log them.

0. Core Principles

  • Be accurate: Record what actually happened, not what you think happened.
  • Be concise: Short, clear descriptions make it easier for Admins to review.
  • Be consistent: Always use the same fields and Case ID format.
  • Be respectful: Treat all users fairly, even when they break rules.

1. Moderation Case Basics (Notion Fields)

Every time you take a moderation action, create or update a case in the Moderation / Cases database in Notion.

1.1 Required Fields

  • Case Title: A quick synopsis of what happened in short form.
    Example: ‚ÄúVoice chat harassment during event‚Äù
  • Case ID: Use the format Case-{Year}-{sequential number}.
    Example: Case-2025-001, Case-2025-002, etc.
  • Users: The user‚Äôs name(s) involved in the case. You can include multiple users in a single case file if needed.
  • Action Taken: All actions you took during the incident:
    • Warnings (verbal or written)
    • Mutes / Kicks / Bans
    • Any escalations to Admins
  • Short Reason: Select from the list of preset violations (e.g. harassment, hate speech, spam, NSFW, etc.). Choose all that apply.
  • Date: The date of the incident (use the date it actually occurred, not when you logged it).
  • Evidence: Screenshots and/or videos:
    • Screenshots can be uploaded directly to this field.
    • Video links should be added here after upload (see evidence section below).
  • Internal Notes: Any additional information you feel may be relevant:
    • Context leading up to the incident.
    • How the user responded to staff.
    • Any history you‚Äôre aware of (if not already logged elsewhere).
  • Reported by: Who Reported the infraction.(Guest/Staff)

1.2 Admin-Only Fields

Leave all remaining fields empty unless you are an Admin or specifically told otherwise. Those fields are for Admin review, final decisions, and long-term tracking.

2. Creating a Case in Notion (Step-by-Step)

  1. Open the Moderation / Cases database in Notion.
  2. Click ‚ÄúNew‚Äù to create a new case entry.
  3. Fill out the required fields listed in Section 1:
    • Case Title
    • Case ID
    • Users
    • Action Taken
    • Short Reason
    • Date
    • Evidence
    • Internal Notes
  4. Add screenshots directly to the Evidence field or paste video message links there (see Section 3).
  5. Double-check spelling, Case ID, and user names for accuracy.
  6. Leave the remaining fields blank for Admin review.

3. Evidence Logging (Screenshots & Video)

Goal: Make sure every case has clear, accessible evidence to back up the actions taken.

3.1 Screenshot Evidence

  • Take clear screenshots that show:
    • Usernames
    • Chat logs or actions
    • Relevant context (not just a cropped insult with no source)
  • In the Notion case, go to the Evidence field.
  • Upload your screenshots directly into this field.
  • Optionally label them with the Case ID for clarity.
    Example: Case-2025-003_screen1.png

3.2 Video Evidence

  • Videos do not go directly into Notion.
  • Upload all video evidence to the #üõÖ„Éªevidence-locker channel in the Staff section of the SkyClub Discord.
  • In your upload message:
    • Mention the Case ID in the message text.
    • Example: ‚ÄúVideo evidence for Case-2025-004‚Äù
  • After the upload completes, right-click the message and choose Copy Message Link.
  • Go back to the Notion case and paste this link into the Evidence field (alongside your screenshots, if any).

3.3 Linking Everything Together

  • Every piece of evidence should clearly reference the Case ID.
  • Admins should be able to open the Notion case and quickly find:
    • Who was involved,
    • What happened,
    • What actions were taken,
    • And the screenshot/video proof.

4. Quick Checklist (Bouncers & Security)

  • Handle the incident calmly and follow SoraSkyClub rules for moderation.
  • Create a new case in the Notion Moderation / Cases database.
  • Set a proper Case Title and Case ID (Case-Year-Number).
  • List all users involved in the Users field.
  • Record all actions taken (warning, mute, kick, ban, escalation).
  • Select appropriate Short Reason violations.
  • Set the correct Date of the incident.
  • Upload screenshots into the Notion Evidence field.
  • Upload videos to #üõÖ„Éªevidence-locker, add the Case ID, then paste the message link into the Evidence field.
  • Add any relevant Internal Notes.
  • Leave all other fields blank for Admin review.

If you are unsure how to classify an incident or what action to take, log what you can, then ping an Admin or Owner in the staff channels for guidance. A well-documented case keeps SoraSkyClub safe, fair, and trusted.
